Branch MASTER = PRODUCTION
Branch DEVELOP = DEVELOPPEMENT 

Personne ne touche à MASTER. Les chefs de projet répondront à des pull request pour merger sur master. 
Tout le développement dit "non production" se fera sur develop


Chaque développeur créer sa branche.
A partir de sa branche, le développeur va créer une branche en fonction de sa tâche :
 - feature
 - bug
 - ... 
Exemple : master > develop > guillian > feature/login


Pour la mise en commun d'un ou de plusieurs développeurs :

 - On récupère le travail (git pull)
 - On merge et on test sur notre branche !!!!
